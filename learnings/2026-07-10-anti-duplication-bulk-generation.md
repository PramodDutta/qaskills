# Bulk AI content that passes mechanical checks but is spun near-duplicate

## Problem

A 100-article batch generated by Codex (gpt-5.5) passed every mechanical check (word count, tables, code blocks, FAQ, valid links, no em dash) but was 63-90% verbatim-identical templated boilerplate across articles: doorway/spun content that would hurt SEO, not help it. Some reused blocks were also topic-mismatched (fuzzing framed as contract testing; ROI code in an OKR article; broken Postgres/NATS/Electron snippets).

## Approach

1. **The audit gate caught what mechanical checks structurally cannot.** Word-count/table/FAQ checks verify STRUCTURE, not UNIQUENESS or CORRECTNESS-of-framing. Eight Opus 4.8 adversarial auditors reading the actual prose found the duplication + wrong framing + broken code. Held the batch, quarantined all 100, shipped nothing.
2. **Root-caused the generation failure, not just the symptom.** Two causes: (a) the spec imposed a FIXED section skeleton ("Why It Matters / Core Concepts / Best Practices / Common Mistakes / FAQ"), which the model turned into a literal template and filled per-wave; (b) each wave was 13 SAME-CLUSTER topics (all WebdriverIO, all Jest), which invited the model to reuse prose within the wave.
3. **Fixed the method and PILOTED before scaling.** New spec: no fixed template, unique topic-specific H2 headings, explicit "do not reuse any sentence/table/example/FAQ across articles", topic-specific worked examples required, varied non-formulaic openings. Plus INTERLEAVE topics so each wave is cross-cluster. Ran 8 diverse pilot articles first, measured, only scaled after proof.
4. **Added a quantitative duplication gate.** Shingle-overlap (8-word Jaccard) between every article pair. Old batch: 63-90% max. New batch: 0% max. This is now a permanent mechanical check, so future duplication is caught without needing the full Opus panel.
5. **Re-audited the regenerated high-risk clusters.** 61/61 PASS on the exact articles that failed before; every prior failure mode (wrong framing, broken code, topic mismatch, false table claims) confirmed fixed.

## Judgment calls

- Did NOT publish the first batch to hit the number. Spun near-duplicate content is negative-value; shipping it to look productive would harm the site.
- Did NOT trust the pilot to bless the whole batch. Piloted the METHOD, then still re-audited the scaled output.
- Fixed the 3 real defects the gates caught (a citation-filter logic bug, an asyncapi wording error, a nested-template-literal escaping bug the build caught) rather than hand-waving them.

## Reusable rule

Mechanical checks prove an article has the right shape, never that it is unique or correct; for bulk AI generation, add a cross-article shingle-overlap gate and an adversarial read of the prose, and when both a rigid-template spec and same-cluster batching are present, the model will spin near-duplicates: give it freedom of structure and interleave diverse topics instead.
